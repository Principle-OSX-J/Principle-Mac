## Overview

Interaction design decisions — animation duration, easing curve, transition choreography, gesture response feel — cannot be made correctly from a description or a static mockup. They must be experienced directly at real speed and real size. <a href="#">Principle</a> provides the environment where these decisions are made by designing and previewing them rather than guessing and specifying them for <a href="#">developer interpretation</a>.

The artboard canvas organises screens as artboards connected through interactive links triggered by taps, swipes, and gesture events. <a href="#">Screen connections</a> with configurable transitions — push, slide, fade, or custom animated — define navigation behaviour across the full multi-screen flow of the application, enabling the complete user journey to be experienced interactively in the prototype. Nested artboards represent components within screens — modals appearing over content, drawers from edges, sheets rising from the bottom — each with their own animation properties composing with the screen-level transitions to create realistic layered interactions.

<a href="#">Scroll layers</a> replicate real application scrolling with configurable direction, content size, paging behaviour, and snap points, making scroll-heavy content interfaces evaluable in the prototype at sufficient fidelity to assess feel. The <a href="#">drag interaction type</a> creates gesture-driven elements that respond to finger position during a drag — enabling swipe-to-dismiss, pull-to-refresh, and card stack interactions with realistic physical responsiveness. <a href="#">Video export</a> captures the animated prototype in action for sharing with stakeholders who need to evaluate interaction design without installing Principle Mirror or accessing design files — making interaction communication practical across teams with mixed tool access. Principle's combination of timeline animation, driver-based gesture response, real-device preview, and <a href="#">video export</a> makes it the most capable interaction prototyping tool available on macOS.

## Key Features

- Timeline-based <a href="#">animation editor</a> with keyframes and easing curves
- <a href="#">Driver-based animation</a> linking motion to scroll and gesture values
- Multi-screen <a href="#">flow design</a> with connectable artboard navigation
- <a href="#">Component states</a> with animated transitions between interactive states
- Real-device <a href="#">preview</a> through Principle Mirror on iPhone and iPad
- <a href="#">Scroll layers</a> with paging and snap point configuration
- Drag <a href="#">interaction type</a> for gesture-responsive element prototyping
- <a href="#">Video export</a> of animated prototypes for stakeholder sharing
- Nested <a href="#">artboard components</a> for modals, drawers, and overlays

## Additional Information

Component state animation enables prototyping of the micro-interactions that most affect perceived UI quality. Button press animations, toggle transitions, and interactive control feedback — 150-300 millisecond transitions — are what users feel most directly, and Principle allows them to be designed and validated before implementation rather than left to developer interpretation.

Video export addresses the practical challenge of sharing prototypes across roles. A video of the prototype communicates interaction design to stakeholders who do not use design tools — without requiring software installation or file access — making design communication practical across teams with mixed technical capabilities.

Principle's focus on motion and interaction rather than static design completeness makes it efficient for the specific phase where interaction behaviour is defined. Importing static designs from Sketch or Figma and adding interaction in Principle keeps each tool at the phase appropriate to its strength.
